# Ambient Agent

Coding agents 能够很好的帮我们编程，甚至诸如 codex 之类的可以当作我们的某种助手，只要下任务给它们就行了。一定程度上，它们的功能已经强大到了似乎无所不能的地步，也就有了很大的想象力，似乎可以接管我们的生活了。

可是我不想让这种商业 agent 来获取我的全部数据。我希望我至少有机会清楚的知道发生了什么、都有哪些数据被利用了、哪部分数据由谁处理…… 为此，我需要一个开源的 agent。

同时，我希望无时无刻都能访问我的 agent，这样我想到什么随时就可以跟它讲。这意味着，我的 agent 在不同的设备上都可以访问，但使用的是同一套（或多套互相同步）的数据。

另外，现在这种单纯基于文字的交互方式太原始了，明明我们在手机上有丰富得多的 GUI。我希望我在不同设备上都能通过丰富的 GUI 来交互，而不是单纯的聊天框。比如，当我问今天天气如何的时候，它应该能给我一个常规天气 app 所具有的所有常见功能。而且我希望这种功能是能够按照我的喜好固定下来（类似一个 skill）。

## Test driven

我希望我们这次的项目遵守测试先行。因为我们大多数代码将会是生成的，所以有一个稳定的测试代码对我们项目的长期运行非常重要。每次实现任何功能之前，先根据设计写出来测试代码明确边界，这样无论我们后面怎么改动，都可以立刻知道是否仍然满足测试要求。所有的测试代码都放在一个统一的文件夹中，并进行必要的层级划分。
